Computer Vision Week7: Depth and 3D

Cues for depth

2D图片中包含着不同的信息可以透露出图片的深度。将多种信息结合起来（如stereo+motion）可以获得更准确的深度信息。

  • Perspective 透视
  • Texture 纹理
  • Object position and occlusion 物体位置和遮挡
  • Object knowledge 物体知识
  • Binocular stereo 双目立体
  • Motion parallax 运动视差

Stereo（立体）

Stereo depth是用于计算视差值（disparity ）的

  • Stereo Pair：由两台在平移位置拍摄的同一物品的两张照片叫做stereo pair。

假设：

两个cameras和baseline都是平行的，两个camera centre在同一高度上，focal lengths f是一样的。

目标：

算出Z的长度

Basic stereo matching algorithm

X为每一个一张图片的像素，扫描X所在的另一张图片的horizontal line，来找到与X最为相似的点X‘。

计算 X与X’的差距，并且计算深度 = ( fB )/(X-X’)

以下两种公式是用来计算两个point的similarity的

Effect of window size

窗口更小，粒度更细，但是noise更多

窗口更大，更平滑的深度，但缺乏细节

Additional constraints

  • Uniqueness: Point应该是1 to 1 matching的
  • Ordering: 对应点的顺序应当保持一致
  • Smoothness: 视差值（disparity ）应该平滑变化

E为最终的similarity formula。

Rectification

上面提到过，image plane需要保持平行，如果image plane不是平行的，则需要使用Rectification来找到能使两个plane变为平行的projective transform。

Single view depth

Supervised depth classification

将深度估测看为一个classification task，通过训练每一张图，来预测每个点距离相机的距离。

Loss Function

由于图片中的深度差异很大，为了减少这些差异，我们经常会做一个log transformation。

场景的平均深度可能会有很大差异（eg 壁橱 和体育场）。为了阻止模型简单地学习平均深度，我们有时需要考虑scale loss function 来应对不同场景。比如体育馆的场景事务深度很长，loss 往往更大。而壁橱的深度较小，需要更精确的loss，所以要做scaling。

由于标注过图片深度的数据（depth map）需要特殊仪器才能获取(LiDAR)，我们也可以用Stereo image pairs来训练模型, 普通相机在拍摄时就可以创建Stereo image pairs。

模型输入：从Stereo pair获取的一张图片

通过学习stereo pair 中另一个的disparity map，来预测另一张图片。

Loss: Appearance loss(原始图和预测图的差异) + Disparity smoothness loss + Left-right consistency loss（两个disparity map的差异）

总结

优点：

使用Machine learning方法我们不需要multiple views 或者stereo camera

缺点：

物品的边缘深度模糊（和edge map结合一起可以让模型表现更好）

模型无法很好地泛化到新的环境中。

Multi-view depth

Multiview Depth 可以解决如下问题

  1. 相机运动 将两个view关联起来的变换（camera translation or rotation）是什么？
  2. 相机参数 （focal ，length）
  3. Scene geometry 已知两张view，求拍摄view相机的3D位置

Camera Calibration 相机校准

相机参数

  1. Intrinsic parameters： 和成像有关系的参数（focal length, optical centre, lens
    distortion)
  2. Extrinsic parameters：和外界有关系的参数（如相机位置，朝向）
  3. Camera calibration 相机校准是一个寻找Intrinsic parameters的方法

在world和image points之间进行转换时，可以使用homogeneous（或projective）coordinates。

每个image point可以用三个数字表达。

第三个数字代表当前与当前image plane的距离。

Camera Calibration 相机校准需要一个校准目标，一个具有已知图案的平面表面，可以通过特征检测(feature detection )轻松detect / track. （例如 棋盘，正方形 or 圆）

需要给棋盘等calibration target s拍摄不同角度姿势的照片来解决intrinsic 和 extrinsic 参数的问题。

  1. 用Planar surfaces 校准， 优点是不需要special calibration target。缺点是检测追踪关键点有困难
  2. 从manufacturer specification中寻找相机参数，优点是不需要计算。缺点是只能给fixed focal length 的相机使用。

Epipolar Geometry

Epipolar geometry是用于预测camera transformation的一种方法。

C, C’, x, x’ 和 X 是共面coplanar，5个点在同一个平面上。

红色线代表Baseline，连接两个camera centers的线。

蓝色点代表Epipole， baseline 与image plane的焦点，另一个camera center的投影

灰色平面代表epipolar plane，包含着baseline的平面

绿色线代表Epipolar line，epipolar平面与image的焦点。

从这张图我们可以看到epipolar并不是一个固定的plane，plane位置取决于X点以及图片位置。

Maths

Epipolar我们可以看成是 原X 是经过K matrix transformation变为 X^\hat XX^ 的。

（也就是说 X=xK2X = \frac{x}{K^2}X=K2x​ ?)

其中T×RT\times RT×R 是cross product。中间将 T×RT\times RT×R 用E（essential matrix）来表示。等于0是因为三者都是coplanar?

当我们不知道K和K‘的时候应当如何求F？

  1. 通过图像中的matching pair point （matching point可以用feature dection-RANSAC）找到可以解释对应关系的F

  1. 8 point algorithm。 需要八个matching point，将F看作linear system解决。需要外增加SVD步骤(singular value decomposition)来确保F的形式正确以便顺利解除。

然而解决linear system只能给出一个scaling factor，想获取真正的距离还需要一个额外的 real world position。如果degrees of freedom太少的话则代表所有点都在一个coplanar上，camera translation 为0 （相当于只是相机rotation了）
